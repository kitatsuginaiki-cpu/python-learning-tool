## 1. プロダクト概要

Python 初心者が使い始め、不自由なく研究（重力マイクロレンズのデータ解析）ができる
レベルまで到達するための、自習用の学習ツール。

ブラウザ内で本物の Python が動く静的 HTML（Pyodide）として配信する。
インストール不要・サーバ不要・利用料ゼロ。無料の静的ホスティング
（GitHub Pages / Cloudflare Pages 等）で `https://` 公開する。

「解説 → 穴埋め演習 → 実データ適用」の3層を1テーマごとに通す、
全9段の到達ラダー形式で学ぶ。

## 2. 背景・課題

MOA 解析セッション（2026-05-20, `M1_mission/`）で詰まったのは Python の文法ではなく
**環境**だった:
- リモート Jupyter の scipy が古く `median_abs_deviation` が使えない
- ローカルに pip / numpy が無い
- Jupyter のセル実行順序による `NameError`

解決したい課題:
- 学習中に「環境が原因のエラー」で止まり、学びが中断される
- ツールの中ではできても、実際の研究環境（Jupyter）では詰まる
- 研究に必要な Python（numpy〜curve_fit〜MCMC）への一本道が無い

→ 最優先要件は「環境で詰まらないこと」。さらに卒業時には環境問題を**自力で直せる**ようにする。

## 3. ターゲットユーザー（ペルソナ）

- **主要利用者**: 自分自身。重力マイクロレンズの解析を始めた大学院生（M1）。
  Python は MOA 解析を一度やり切った程度。基礎は穴あき、研究直結部は実戦経験あり。
- **想定の広がり**: 同様の Python 初心者（研究室の後輩など）。ただし現時点では多人数共有は
  スコープ外（セクション6）。
- 利用環境: PC のモダンブラウザ。ネット接続あり（初回ロード時）。

## 4. 機能要件

機能要件は **MVP＝縦スライス**（エンジン＋段1を完全実装）を Must とし、
残りの段はコンテンツ追加で順次拡張する（Should）。

### Must（MVP・縦スライス）
- **ブラウザ内 Python 実行環境**: Pyodide を組み込み、numpy / scipy / pandas /
  matplotlib を利用可能にする。初回ロード中はプログレス表示を出す。
- **レッスンの3層構造 UI**: 1テーマごとに「最小解説」「穴埋めコードエディタ」
  「実行ボタン」を提供する。
- **`assert` ベースの自動採点**: 学習者のコードを実行し、結果（変数・出力）を検証して
  合否を表示する。コードの書き方は縛らない。
- **失敗時の固定ヒント表示**（AI は使わない）。ヒントには **traceback（エラーメッセージ）
  の読み方** を織り込む。エラー読解は特定の段ではなく**全段共通の横串スキル**として扱う。
- **1セル完結の演習**: 各演習が import を含み単独実行可能。セル実行順による
  `NameError` を構造的に排除する。
- **段1「numpy」を完全実装**: 複数の穴埋め演習＋`M1mission` の `.phot.txt` を使う
  卒業課題。これが MVP の縦スライスで、エンジン全機能（Pyodide / numpy /
  `assert` 採点 / ファイルのドラッグ＆ドロップ）を一巡して検証する。

### Should（MVP後にコンテンツ拡張）
- **段0「Python素の基礎」**: 変数・型・list・dict・for/if・関数。
  段0は複数レッスン群＝最大のコンテンツであり、スコープを慎重に設計する（セクション8）。
- **段2〜6**: pandas / matplotlib / `curve_fit` / フィット評価 / MCMC。
- **段7「Jupyter運用」**: セル実行順・カーネル。実環境への橋渡し。
- **段8「実環境を整える力」**: venv・pip・パッケージ版数。
  （エラー読解は Must の横串へ移したため、段8は環境構築に純化する。）
- **進捗の保存・再開**: localStorage に学習進捗を保存。
- **emcee の実行時インストール**: MCMC レッスンで `micropip.install("emcee")`。
- **プログレッシブ・ブランク**: 段0〜3 は手厚い穴埋め、段4以降は空白を広げる。

### Could（将来）
- AI ヒント / AI 採点（Claude API、課金前提。固定ヒントで足りない場合のみ検討）。
- 進捗ダッシュボード・修了バッジ。
- 自分の研究データを汎用的に読み込む機能（MOA 以外のフォーマット）。
- ダーク / ライトテーマ。
- 後輩にも使わせる段階での SaaS 化（進捗のサーバ保存・共有）。

## 5. 非機能要件

- **配信**: 無料の静的ホスティング（GitHub Pages / Cloudflare Pages / Netlify 等）で
  `https://` 公開。`file://` は CORS で Pyodide が失敗しうるため非対応。
  GitHub Pages は無料だと公開リポジトリが前提のため、**ソースを非公開にしたい場合は
  Cloudflare Pages / Netlify を選ぶ**（private ソースでも無料で静的公開可）。
- **初回ロード**: numpy/scipy/pandas/matplotlib で数十MB・十数秒を許容。
  プログレス表示必須。2回目以降はブラウザキャッシュで高速化。
- **オフライン**: 完全オフライン前提にはしない。scipy/pandas 等は初回 `import` 時に
  DL されるため、**必要なパッケージを一度 import し終えた後のみ**オフライン動作可。
- **コスト**: ゼロ。バックエンド・サーバ・有料 API を使わない。
- **対応ブラウザ**: Chrome / Edge / Firefox / Safari の最新版。
- **セキュリティ・プライバシー**: 個人情報を扱わない。学習者のデータ（進捗・読み込んだ
  `.phot.txt`）はブラウザ内（localStorage / 仮想 FS）のみで、外部送信しない。
- **保守性**: 単一プロジェクトとして GitHub リポジトリで管理。レッスンは追加しやすい
  データ構造（後続の「レッスン設計」工程で設計）。

## 6. スコープ外（今回は作らない）

- バックエンド / サーバ / ユーザー認証。
- 多人数共有、進捗のサーバ保存・共有。
- AI による自動採点・ヒント（現時点。Could に留める）。
- Python 以外の言語の学習。
- 発表スライドや解析レポートの自動生成。

## 7. 成功の定義（完成の基準）

### MVP 完成の基準
- 初心者が **段1（numpy）の全レッスンをツール内で自力完了**できる。
- 段1の卒業課題で、自分の `.phot.txt` を読み込み numpy で基本統計
  （median / MAD 等）を算出できる。
- ツール利用中に「環境が原因のエラー」が発生しない（Pyodide で根治）。
- インストール作業ゼロで、URL を開くだけで学習を開始できる。

### 最終形（全段）の基準
- 完全初心者が段0から始め、**段6の卒業課題（自分の MOA データで `curve_fit`）を
  自力で完了**できる。
- 段7・8修了後、ユーザーが実際の Jupyter で `median_abs_deviation` の版数問題の
  ような詰まりを**自力で診断・解決**できる。

## 8. 主要リスクと前提

- **コンテンツ作業量**: 本プロジェクトの作業の大半（およそ8割）はレッスン執筆
  （9段 × 3層 × 複数問）。Pyodide エンジンの実装は2割程度。
  → MVP を段1の縦スライスに絞り、残りの段はコンテンツ追加で段階拡張する。
- **段0のスコープ**: 段0（基礎）を絞りすぎると初心者が numpy 以降で詰まり、
  「環境で詰まらない」というツールの存在意義を文法面で裏切る。段0は複数レッスン群
  として、何を入れ何を切るかを専用に設計する。
- **Pyodide の制約**: matplotlib の描画、emcee の実行時 DL など、ブラウザ実行特有の
  挙動がある。STEP2（技術スタック）で実装前に検証する。
