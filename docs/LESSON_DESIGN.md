## 1. コンテンツ・スキーマ（レッスンのデータ構造）

レッスンは「追加しやすいデータ」として持つ（PRD 非機能・保守性）。
データ形式（JSON / JS モジュール / 段ごとのファイル分割）は STEP2・STEP4 で確定。
ここでは**構造**を定める。

### 段（Stage）
| フィールド | 内容 |
|---|---|
| `id` | 例 `"stage-1"` |
| `number` | 段番号（0〜8） |
| `title` | 例 `"numpy — 配列で測定データを扱う"` |
| `intro` | 段の導入文（なぜこの段か、1段落） |
| `lessons` | レッスンの配列 |
| `capstone` | 実データ卒業課題（Exercise 1問） |

### レッスン（Lesson）
| フィールド | 内容 |
|---|---|
| `id` | 例 `"stage-1-lesson-3"` |
| `title` | 例 `"ブール索引 — 悪い測定点を外す"` |
| `explanation` | 最小解説（Markdown, 5〜10行。末尾に「研究での出番」を一文） |
| `exercises` | 穴埋め演習の配列（複数） |

### 演習（Exercise）— 3層構造の2層目・3層目の最小単位
| フィールド | 内容 |
|---|---|
| `id` | 例 `"stage-1-lesson-3-ex-2"` |
| `prompt` | 問題文（何をすればよいか） |
| `starterCode` | 穴あきコード。空欄は `____` で表す。**import を含み1セル完結** |
| `solutionCode` | 模範解答。`____` を埋めた実行可能な完全コード（「答えを見る」用） |
| `test` | 採点コード（`assert` 群。ユーザーコード実行後、同じ名前空間で実行） |
| `hints` | 段階的な内容ヒント（弱→強）の配列 |
| `blankLevel` | `guided` / `partial` / `blank`（プログレッシブ・ブランク用） |

### エラー読解ガイド（段横断・共通）
PRD の見直し3「エラー読解は全段横串」を実装するための共通データ。
```
errorGuide = {
  "NameError":  "未定義の名前。変数のスペルミス or import 漏れを疑う…",
  "TypeError":  "型の不一致。配列とリストの混在、関数の引数違いを疑う…",
  "IndexError": "範囲外アクセス。配列の長さ（.shape / len）を確認…",
  "ValueError": "値が不正。形の合わない配列演算、変換できない文字列を疑う…",
  ...
}
```
例外発生時、エンジンは例外型に対応するガイドを traceback と並べて表示する。

## 2. 採点・ヒント表示フロー

1. ユーザーが `starterCode` の `____` を埋めて実行。
2. エンジンが Pyodide で**ユーザーコードを名前空間 `ns` で実行**。
3. 例外が出たら → traceback を整形表示 ＋ `errorGuide[例外型]` を表示
   （該当 `hints` の弱いものも併記）。
4. 例外なし → 同じ `ns` で `test` を実行。
5. `test` の `assert` 成功 → **合格**。`AssertionError` → `hints` を段階的に開示。

ポイント: 採点は**結果（変数・出力）を検証**し、コードの書き方は縛らない。

## 3. 段1「numpy」執筆計画（MVP の縦スライス）

研究（マイクロレンズ測光データ `.phot.txt`）に直結する numpy だけに絞る。
網羅的な numpy 入門にはしない。

| レッスン | テーマ | 主な内容 | 研究での出番 |
|---|---|---|---|
| L1-1 | 配列を作る | `np.array`, `np.arange`, `.shape`, `.dtype` | 測定列を配列にする |
| L1-2 | 要素アクセスとスライス | `arr[0]`, `arr[2:5]`, `arr[-1]` | 一部期間を切り出す |
| L1-3 | 配列の演算 | 四則演算・ブロードキャスト・ベクトル化 | flux 同士の差、定数倍 |
| L1-4 | ブール索引 | マスク作成, `arr[arr>0]`, `&` `\|` | 誤差の大きい点を除外 |
| L1-5 | 統計関数 | `mean` `std` `np.median`, MAD の計算 | 外れ値検出（MOA で詰まった所） |
| 卒業課題 | 実データ適用 | `np.loadtxt` で `.phot.txt` を読む → ブール索引で悪点除外 → median/MAD | MOA 解析の最初の一歩 |

- 各レッスン ＝ 解説（5〜10行）＋ 穴埋め演習 2〜3問。
- 段1 合計 ＝ 約10〜15演習 ＋ 卒業課題1問。これが MVP の完成スコープ。
- `blankLevel` は段1なので全体 `guided`（手厚い穴埋め）寄り。
- 卒業課題は `.phot.txt` をドラッグ＆ドロップ → Pyodide 仮想 FS → `np.loadtxt`。
  pandas は段2なので、段1は numpy だけで完結させる。

## 4. スキーマ検証用サンプル演習（L1-4 ブール索引）

スキーマが機能することを示す、完全に埋めた1問。

```
id:         "stage-1-lesson-4-ex-2"
blankLevel: "guided"
prompt:     配列 mag のうち、誤差 err が 0.1 より小さい「良い測定点」だけを
            取り出して good に入れよう。
```

`starterCode`:
```python
import numpy as np
mag = np.array([18.2, 18.0, 17.9, 18.1, 18.3])
err = np.array([0.05, 0.30, 0.08, 0.40, 0.06])

mask = ____          # err が 0.1 より小さいか の True/False 配列
good = mag[____]     # mask を使って mag を絞り込む
print(good)
```

`solutionCode`（`____` を埋めた実行可能な完全コード）:
```python
import numpy as np
mag = np.array([18.2, 18.0, 17.9, 18.1, 18.3])
err = np.array([0.05, 0.30, 0.08, 0.40, 0.06])

mask = err < 0.1
good = mag[mask]
print(good)
```

`test`:
```python
assert good.tolist() == [18.2, 17.9, 18.3], f"good が違います: {good}"
```

`hints`:
1. ブール索引はまず True/False の配列（マスク）を作る。`err < 0.1` がそれ。
2. `mag[mask]` のように配列へ True/False 配列を渡すと、True の位置だけ残る。
3. 答え: `mask = err < 0.1` ／ `good = mag[mask]`
