## 2. 背景・動機

MOA解析セッション（2026-05-20, `M1_mission/` 参照）で詰まったのは Python そのものより
**環境**だった:
- リモートJupyter の scipy が古く `median_abs_deviation` が使えない
- ローカルに pip / numpy が無い
- Jupyter のセル実行順序による `NameError`

→ 学習ツールは「環境で詰まらないこと」を最優先要件にする。

## 3. 配信形態の比較と暫定結論

| | SaaS | HTML(静的)+Pyodide | デスクトップアプリ |
|---|---|---|---|
| 導入 | URL開くだけ | ファイル/GitHub PagesでURL | インストール必要 |
| Python実行 | サーバ側 | **ブラウザ内(Pyodide), 導入ゼロ** | 同梱Python |
| 維持コスト | サーバ代・保守が継続 | **ほぼゼロ** | 配布・更新が面倒 |
| オフライン | 不可 | 可 | 可 |
| 多人数共有 | 得意 | 不得意 | 不得意 |

**暫定結論: HTML（静的）＋ Pyodide を推奨。**
- 目的は「自分の学習」で多人数共有は不要 → SaaSの重い土台（バックエンド/認証/サーバ代）は過剰。
- Pyodide はブラウザ内に numpy/scipy/pandas/matplotlib を持つ → インストールゼロで Python が動く
  → 環境問題の根治になる。
- 教材を実際の研究（`M1mission` 解析: curve_fit・残差・MCMC）そのものにできる。
- SaaS化は「研究室の後輩にも使わせる」「進捗をサーバ保存・共有」が主目的になった段階で
  静的HTML → SaaS へ昇格させればよい（後からでも自然な道）。

## 7. 深掘り結果（2026-05-20 続きセッション）

### 7-1. 配信形態 → GitHub Pages 上の静的HTML＋Pyodide で確定
- numpy/scipy/pandas/matplotlib は Pyodide が公式バンドル。**emcee は別** →
  実行時に `micropip.install("emcee")` で追加（MCMCレッスンのみ追加ロードが入る）。
- **`file://` で開くと CORS で Pyodide が失敗しうる** → 「環境で詰まらない」要件に反する。
  GitHub Pages（`https://`）配信を正式形態とする。維持コストはゼロのまま。
- 初回ロードは数十MB・十数秒 → **ロード中プログレス表示を必須要件にする**。2回目以降はキャッシュ。
- 実データ（`.phot.txt`）はドラッグ＆ドロップ → Pyodide 仮想FSに書き込み。進捗は localStorage。

### 7-2. 学習スタイル → 穴埋め主軸の「3層構造」で確定
「軸を1つ選ぶ」のではなく、1テーマを3層で通す（スキャフォールディング）。
1. **ねらい＋最小解説**（5〜10行、なぜ微lensing解析で要るかを一言）
2. **穴埋め演習**（複数問・`assert`で自動採点・失敗時テーマ特化ヒント）← 主軸
3. **実データ適用**（`M1mission`の`.phot.txt`を使う卒業課題1問）

設計上の要点:
- 各演習を **1セルで完結**（import毎回含む）→ セル実行順による NameError を構造的に排除。
- 「Jupyterのカーネル／セル順」自体を1レッスンとして明示収録（セクション5最終項）。
- ユーザーはMOA解析の実務経験あり → 新概念は穴埋め、復習回は白紙記述と
  **段階的に空白を広げる**（プログレッシブ・ブランク）。

### 7-4. 収録範囲 → 「初心者→研究遂行レベル」の到達ラダーで確定
ツールの目的を確定: **Python初心者が使い始め、不自由なく研究できるレベルまで到達する**。
よって研究直結に絞らず、入口から研究遂行までの一本道を全段収録する。
各段は3層構造（解説→穴埋め→実データ）。

| 段 | テーマ | 位置づけ |
|---|---|---|
| 0 | Python素の基礎（変数・型・list・dict・for/if・関数） | 助走。研究で使う形に絞り短く |
| 1 | numpy（配列・ブール索引・統計 median/MAD） | 研究直結 |
| 2 | pandas（read_csv・sep/header・列操作） | 研究直結 |
| 3 | matplotlib（errorbar・subplots・残差） | 研究直結 |
| 4 | scipy.optimize.curve_fit（モデル・初期値・bounds・共分散） | 研究直結 |
| 5 | フィット評価（残差・χ²・zスコア） | 研究直結 |
| 6 | MCMC（emcee・corner plot） | 研究直結 |
| 7 | Jupyter運用（セル実行順・カーネル） | 実環境への橋渡し |
| 8 | 実環境を整える力（venv/pip・パッケージ版数・エラー読解） | 研究遂行力。卒業段 |

設計方針:
- 段0〜3は「速い助走」。網羅的Python入門にせず、研究に出る文法だけ。
- 段7・8がゴール到達の鍵。ツールは環境問題を肩代わりするが、卒業時は自分で直せるように。
  段8は MOAセッションの `median_abs_deviation`（古いscipy）問題を教材化する。
- プログレッシブ・ブランク: 段0〜3は手厚い穴埋め、段4以降は空白を広げる。
